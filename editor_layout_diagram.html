问题根因分析

Widget 树层级： Scaffold → Stack → Column[Header, Expanded(Canvas), Panel?] + DraggableScrollableSheet(overlay)

触发条件：选中图片 → EditorState.single → LayoutSelectionPanel(280px) 从 Column 中移除 → Expanded 画布区域 高度增加 ~280px → 但 DataDrivenCanvas 的 _needsRecenter = false（因为 canvasConfig 和 imageBlocks 未变） → 画布保持在旧位置和旧缩放 → 底部出现多余空间（显示画布背景色=黑色）

修复方案：在 Single 状态下，用一个 280px 的空白占位 替代 LayoutSelectionPanel，保持 Column 布局一致。Expanded 画布视口高度不变， 画布位置不需要重新计算，从根本上消除多余空间。DraggableScrollableSheet 仍作为 Stack overlay 显示在占位区上方。
